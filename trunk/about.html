Web Protocols

Like most AJAX libraries protocols.js attempts to bridge the compatibility gap between browsers, although on a smaller set of interfaces, providing conveniences to leverage a stable stack of web protocols: HTTP, HTML, CSS and JSON.

Call it SOA, ROA or - more appropriately - a Representational State Transfer architecture, the design of a web applications is constrained first by the protocols they implement. Your mileage of application server systems, languages or APIs may vary, if their protocols do not.

To deliver an application that meets its entreprise requirements on time and budget, protocols.js supports a model driven architecture where blueprints are written directly using the text protocols applied. Starting with JSON for object modeling, CSS and HTML for presentation, HTTP for integration. This library is designed to leverage web protocols, allowing analysts, programmers and web designers work independantly from each other but as efficiently as if they were one.

For JavaScript

The 700th implementation of a Lisp interpreter in C clothing does not suck.

So, I did not bend JavaScript like Ruby (Prototype), Python (Mochikit) or Java (Dojo).

Nor did I try to reimplement yet another framework for throwable web applications. If you need one, use jQuery, Ext or both. This library was writen to get entreprise web applications developers where less is more, now and in long run.

The absence of a ton of abstractions or another language emulation makes protocol.js easier to learn and ensures that its applications are reliable, small and fast enough.

Entreprise Applications

I use protocols.js to develop model-driven user interfaces for entreprise web applications.

Making an application look good in the eyes of all its users demands to perform it quickly and without fault everywhere. Dressing up ad-hoc HTML templates and sleek CSS styles on every browsers supported can be done separately, concurrently and without changes to JavaScript programs.

Note that positionning, effects, drag & drop or most browser bugs are not supported by this library. Fancy visual effects in JavaScript are expensive and tricky to run accross plateforms. Drag and drop has allways been a gimmick with many pitfalls for its users and GUI widgets in a web browser are non-senses.

Entreprise applications should not be developped on broken programming interfaces or depend all the HTML and CSS incompatibilies between web browsers. To deliver get great user interfaces everywhere as quickly as possible and at the best available price the best possible course of action is to distribute the effort and distribute the work between many web designers, up to one by software and hardware plateform supported.
