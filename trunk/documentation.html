protocols.js

Documentation

This library provides a collection of conveniences that implement the cross-browser boilerplate code required by all Web 2.0 applications:

  1. Functions
    Four general conveniences for JavaScript.
  2. HTTP
    A statefull HTTP client that provides reliable support accross browsers for asynchronous requests for text responses, with timeouts and without redundant submission.
  3. HTML
    Update, replace and insert HTML markup; Add or remove CSS classes; Serialize a query string from HTML form input values; Escape text as valid CDATA.
  4. JSON
    Encode and decode JSON strings; GET or POST requests for a JSON response with URLencoded or JSON payload; Template JSON objects into HTML markup; Update, replace and insert in an HTML page the templated JSON responses.

In as far as possible, each interface is documented with the synopsis of an online test case. Such active synopsis change their left border color from grey to orange when hovered and execute their code when clicked. 

alert('synopsis');

Usually, synopsis yield a simple message box with the result of a function, method or callback.

Interfaces that interact with the HTML page also use the orange color on hover to highlight where the page is active, as seen in the "Hello World!" test case.

About This Implementation

JavaScript has a bad reputation, but this 700th implementation of a Lisp interpreter in Java clothing does not suck.

So, I did not bend JavaScript like Ruby (prototype), Python (mochikit) or Java (Dojo).

Neither did I try to reimplement yet another framework for glamourous throwable web applications. This library was writen for profitable investments, for entreprise web applications development to get where less is more now and in long run.

This library has been tested with stable versions of Firefox and Internet Explorer only, does not do effects or drag-n-drop and weights less than 4KB once minified and compressed.

Like most JavaScript libraries protocols.js attempts to bridge the compatibility gap between browsers, although on a smaller set of interfaces and with a special focus on names.

With a tall stack made of JSON, HTTP, HTML and CSS to apply from the client, it does not hurt to have simple JavaScript API for these protocols and proper names.
