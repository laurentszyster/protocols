A JavaScript library to apply web protocols

Synopsis

Here is the usual "Hello World!" suspect's sources, with linked dependencies: 

<html>
  <head>
    <title>hello protocols!</title>
    <link href="protocols.css" type="text/css" rel="stylesheet" ></link>
    <script src="protocols.js" type="text/javascript" ></script>
  </head>
  <body>
    <a 
        href="#" 
        onclick="JSON.GET('hello-world.js', null, JSON.update());"
        >hello</a>
    <span id="who" ><em class="who" >...</em></span>
  </body>
</html>
hello ...

This first test case does the simplest possible thing required from a web 2.0 client application: load a JSON object from its server and update the HTML page content with its properties, mapping names in the response to elements in the DOM.

Applications

This library is designed to support complex and large entreprise web 2.0 applications.

It is simple and small, quick to learn, fast to run and safe to depend on.

As the simplicity of the sources above illustrate, protocols.js leverage better conventions and provides practical conveniences. Note also how the HTML page above can acts as a substitute of traditionnal server-side template, backed for demonstration purposes by a static JSON resource instead of an actual service controller (hello-world.js).

I use protocols.js to develop model-driven user interfaces for web application services programmed in JavaScript, Java or CPython. It can be used to integrate services built with other development tools and running on a different computer.

Your mileage of application server systems, languages or APIs may vary.
