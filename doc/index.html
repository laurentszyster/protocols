If the network is the computer then protocols are its application programming interfaces.

Great JavaScript applications of HTTP, HTML, CSS and JSON are written as a web of URLs, hypertext markup, cascading styles and JavaScript object notations.

Protocols is a small library of JavaScript conveniences to apply the stable stack of web protocols for common browsers, with less code and more lever on protocols.

It combines the pearls of blog wisdom, some bits of Crockford's sagacity, a great deal of Jock Slocum's selector and two cents of Prototype's tips.

Its sources are kept small by avoiding the temptation of yet another GUI library in a browser.

Web protocols are meant to decouple visual design (CSS) from the page flows (HTML), the interactivity automations (JavaScript), the network resource controllers (HTTP) and the application's object models (JSON).

I won't bring back the fat GUI client zombies back from the 80's.
